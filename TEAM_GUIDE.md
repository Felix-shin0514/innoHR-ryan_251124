# 🔰 팀 협업 가이드 (초보자용)

안녕하세요! 이 문서는 개발이 낯선 팀원분들도 **GitHub**를 쉽게 사용하여 협업할 수 있도록 만든 가이드입니다.

---

## 1. GitHub가 뭔가요? 🤔
**"우리 팀의 공유 클라우드 폴더"**라고 생각하면 쉽습니다.
- **내 컴퓨터(로컬)**에서 작업한 파일을 **GitHub(원격 저장소)**에 올리면, 다른 팀원들이 그걸 내려받아 볼 수 있습니다.
- 반대로, 다른 팀원이 올린 작업을 내가 내려받을 수도 있습니다.

---

## 2. 작업 시작 전 준비물 🛠️
1. **VS Code (Visual Studio Code)**: 코드를 작성하는 프로그램입니다.
2. **Git**: 파일을 주고받게 해주는 배달부 프로그램입니다.

---

## 3. 협업 순서 (Workflow) 🚀
우리는 **"안전하게 작업하기"** 위해 아래 5단계를 꼭 지켜야 합니다.

### 1단계: 최신 파일 받아오기 (출근 도장)
작업을 시작하기 전에, 다른 팀원이 해놓은 최신 작업을 내 컴퓨터로 가져와야 합니다.
> **명령어 입력 방법:** VS Code 상단 메뉴 `Terminal` -> `New Terminal`을 누르고 아래 명령어를 치세요.

```bash
git checkout main      # 1. 메인 작업대(Main)로 이동
git pull origin main   # 2. 최신 파일 당겨오기(Download)
```

### 2단계: 나만의 작업 공간 만들기 (책상 정리)
`main`은 우리 팀의 최종 완성본입니다. 여기서 바로 작업하다가 실수하면 큰일 나겠죠?
그래서 **나만의 작업 공간(브랜치)**을 따로 만들어서 거기서 작업합니다.

```bash
# git checkout -b [작업할_내용_영어이름]
# 예시: 로그인 화면을 만든다면?
git checkout -b feature/login-page
```

### 3단계: 작업하고 저장하기 (중간 저장)
코드를 수정하고 나면, 이 변경 사항을 **"확정(Commit)"** 해야 합니다.
단순히 `Ctrl+S`로 파일만 저장하는 게 아니라, **"이 작업을 기록으로 남기겠다"**고 선언하는 과정입니다.

```bash
git add .                        # 1. 변경된 모든 파일을 장바구니에 담기
git commit -m "로그인 버튼 색상 변경"  # 2. 어떤 작업을 했는지 메모와 함께 포장하기
```
> **Tip:** 메시지(`"..."`)는 한글로 구체적으로 적어주세요! 예: "메인 화면 로고 크기 수정"

### 4단계: GitHub에 올리기 (업로드)
내 컴퓨터에만 저장된 작업을 팀원들이 볼 수 있게 **GitHub(원격 저장소)**로 올립니다.

```bash
# git push origin [아까_만든_내_작업공간_이름]
git push origin feature/login-page
```

### 5단계: 합치기 요청 (결재 올리기)
이제 내 작업을 `main`(최종 완성본)에 합쳐달라고 요청해야 합니다. 이것을 **PR(Pull Request)**이라고 부릅니다.

1. [GitHub 저장소 페이지](https://github.com/Felix-shin0514/innoHR-ryan_251124)로 이동합니다.
2. 상단에 노란색 박스로 **"Compare & pull request"** 버튼이 생겼을 겁니다. 클릭하세요.
3. 내용을 적고 **Create Pull Request** 녹색 버튼을 누르면 끝!
4. 이제 팀장님이나 다른 팀원이 확인하고 `Merge` 버튼을 눌러주면 합쳐집니다.

---

## 4. 자주 쓰는 명령어 요약표 📝

| 상황 | 명령어 | 의미 |
|---|---|---|
| **시작할 때** | `git pull origin main` | "최신 파일 주세요!" |
| **작업 시작** | `git checkout -b [이름]` | "새 작업 공간 만들게요!" |
| **저장할 때** | `git add .` <br> `git commit -m "메모"` | "작업 내용 포장할게요!" |
| **올릴 때** | `git push origin [이름]` | "GitHub에 업로드할게요!" |
| **확인할 때** | `git status` | "지금 내 상태가 어때요?" |

---

## 5. 비상 상황 대처법 🚨

**Q. `git pull`을 했는데 에러가 나요!**
A. 내 컴퓨터에서 수정한 내용과 GitHub의 내용이 충돌해서 그렇습니다. 당황하지 말고 팀장님을 부르세요!

**Q. 어떤 브랜치에 있는지 모르겠어요.**
A. `git branch`를 치면 현재 내 위치가 나옵니다. 별표(*)가 붙은 게 현재 위치입니다.
